Generating Reports and Javadocs - Applicable to Windows OS only!!!!!

  1. Running Scripts with WScript.exe

    1. WScript.exe (usually in Windows/System32 folder) is a Windows Script Host that enables execution of scripts from Windows. To run a script, do the following:
    2. Open a new command prompt and navigate to the folder where the script file (to be executed) is located.
    3. At the command prompt, type "WScript.exe XYZ", where XYZ refers to the script file to be exectuted.
    4. Example: WScript.exe protexpress_create_value_domain_report.js

  2. Generating the Value Domain Report

    1. Download the protexpress_create_value_domain_report.js file from SVN.
    2. Open the Javascript file in a text editor, and modify the following variables:
      1. eaFilePath: The path to the EA Model File (eg: protExpressGrid.eap).
      2. csvFilePath: Name and location of the report file to be created (eg: 08. protexpress_value_domain_report.csv).
    3. Execute the script as described above to generate the report file in CSV format.

  3. Generating Javadocs for the model

    1. Download the protexpress_create_pojos_from_model.js file from SVN.
    2. Open the Javascript file in a text editor, and modify the following variables:
      1. eaFilePath: The path to the EA Model File (eg: protExpressGrid.eap).
      2. GENERATED_JAVA_DIR: Location where the java classes will be generated.
    3. Execute the script as described above to the java classes. This will generate the java source files in the folder specified by the variable GENERATED_JAVA_DIR.
    4. Browse to the folder specified by GENERATED_JAVA_DIR, open a new command prompt and execute the following command:
      1. javadoc -d javadocs -public -sourcepath . -subpackages gov
    5. If the above command executes successfullly, a new subfolder named javadocs (containing the javadocs) will be created under the folder specified by GENERATED_JAVA_DIR.
