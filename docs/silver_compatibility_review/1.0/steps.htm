1. Generate a UML Model in Enterprise Architect

    1. Create a New Project.
    2. In the Project Browser view, add the Java packages that will be used.
    3. In Logical View->Logical Model, create the package structure.
    4. Add the appropriate classes, create the diagrams. For more information, refer to the ....
    5. Ensure that all classes and class attributes have a description. This description will be when the model is loaded (as an XMI file) into SIW.

  2. Export the XMI file from EA

    1. Select Logical View in the project browser.
    2. Check Enable Full EA Roundtrip.
    3. Export the model to an XMI File, and save it to the local disk.

  3. Annotate the model (xmi file)

    1. Start Semantic Integration Workbench (SIW).
    2. Load the XMI file into SIW, and go through all the steps..
    3. Ensure that:
      1. Each attribute of every class has a concept code mapped. SIW attempts to match the attributes to concept codes. In certain cases, no concept code may be mapped by default, or multiple concept codes could be mapped to a single attribute. Review and make changes as appropriate (add concept codes, delete mapped concept codes etc..)
      2. Classes and attributes are mapped correctly, and concepts are semantically correct.
    4. Once changes have been done, save the XMI file.

  4. Import the annotated XMI file into EA project

    1. Open the EA project in Enterprise Architect.
    2. Import the annotated XMI file (from the previous step) into EA. NOTE: This will overwrite your existing model..
    3. At this point, the EA project file has all the annotations. Annotations for classes and attributes can be viewed as tagged values (Hit Ctrl+Shirt+6).
    4. You can make changes to the model, export to XMI, run it through SIW and so on...till the model is annotated as much as possible.

  5. Submit the model to EVS Team for review, and load to caDSR

    1. Submit Compatibility Review Submission artifacts/package - This requires submission of the following artifacts:
      1. Powerpoint presentation - Provides an overview of protExpress, brief summary of the functionality and architecture.
      2. Vision Document
      3. Software Architecture Document.
      4. UML Model - This is a copy of published protExpress model(protExpressGrid.eap)
      5. Semantically Annotated XMI File - This should be the most recent copy of the XMI file submitted to caDSR for loading.
      6. SIW Report/Error Log - SIW no longer produces a report, and hence this need not be included. The error log can be generated from SIW though.
      7. UML Loader Checklist
      8. Value Domain Report - Can be generated via a script here.
      9. Vocabulary Report - Skipped
      10. Standards Report
      11. Full CDE Use Report
      12. Class Documentation/JavaDocs for the published model - Can be generated via a script here.
      13. Adpoter Test Log - Skipped
      14. Pointers to Source Code - A text document with URLs to the API and Grid Service source code in SVN.
      15. Test Approach Document - Skipped
      16. Use Case Documentation
