Initiated by a protExpress user, this use-case should be able to:
1. Read and parse data entered by the user in the Graphical User Interface;
2. Format the data entered via the GUI into a custom format (tab delimited and/or a custom XML format - .protx?) and/or into other formats supported by the system. The current plans include supporting xar.xml and Pride XML Formats.
3. Ability to read and parse an existing file (xar.xml format, Pride XML and/or the custom Format) and validate it.
4. Experiment information will be annotated using a supported ontology.
