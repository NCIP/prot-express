Working with Protocol Actions
A protocol action defines the sequence of protocols that you want to apply to an experiment. A sequence is typically a number series; sequence numbers do not have to be unique. For example, when the outputs of multiple protocols are inputs to an experiment, those outputs may all have the same sequence number.
The output of each protocol in the sequence defined by the protocol action is the input to the next protocol in the sequence. You define the initial input protocol to the experiment in the Protocol Application Inputs tab. For more information, see Working with Protocol Application Inputs.
